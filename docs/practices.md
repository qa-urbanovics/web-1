# Практики QA × DevOps

## Пайплайн как продукт
!!! tip "Цель"
    CI должен быть быстрым, детерминированным и объяснимым.

- Разделение: lint → unit → contract → API → UI → e2e
- Параллелизация и quarantine для flaky
- Артефакты: логи, видео, скриншоты, trace

## “Definition of Done” для автотестов
- нет флейка / известные причины флейка закрыты
- понятные сообщения об ошибке
- понятные предусловия и тест-данные
